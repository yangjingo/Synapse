# BLOG-STYLE-CN: Why.J 研究与工程写作元审美

## 1. 写作 DNA

### OpenAI 风格 (数据驱动)

- **结构**: 挑战 → 解决方案 → 指标。
- **语调**: 极简、专业、高权威感。

### Anthropic 风格 (严谨)

- **结构**: 问题 → 详细方法 → 局限性 → 讨论。
- **语调**: 深度解释、对话式但保持学术感。

### Karpathy 风格 (直觉)

- **结构**: 钩子 (Hook) → 分步“配方” (Recipe) → 提示与技巧。
- **语调**: 强个人风格、“发生了什么”的代码块。

## 3. Synapse 创作五律 (The Synapse Ethos)

这是一种典型的**“硬核工程师随笔”**风格。它是写给那些“深夜还在调参数、看日志、对满天飞的新名词冷冷一笑”的技术同类看的。

### I. 祛魅原则 (Anti-Hype / No-Bullshit)
- **核心：** 拒绝名词泡沫，直击工程本质。
- **执行：** 遇到高大上的新词（如 Agentic Workflow），自动转译为具体的工程实现（如“带状态机的循环逻辑”）。不卖关子，不讲宏大叙事，首段即进入战场。

### II. 伤痕美学 (Failure-Driven / Practical Realism)
- **核心：** 失败的细节比成功的结论更有信息熵。
- **执行：** 详细记录“因为某某限制导致某某方案由于某某原因失败”的过程。用“坑”来定义“路”，让内容带有真实的粗糙感和现场感。

### III. 古典基座 (Foundational Logic / Classic Modernism)
- **核心：** 前沿 AI 问题的解药往往藏在经典软件工程里。
- **执行：** 强调解耦、约束、确定性。在讨论神经网络或大模型时，始终保持架构师的视角，寻找与单一职责（SRP）或关注点分离（SoC）的映射关系。

### IV. 机械精度 (Terminological Precision / Geek Rationalism)
- **核心：** 词语是思想的刻度。
- **执行：** 使用具有机械感和结构感的动词（如：注入、暴露、编排、路由、原子化）。排版推崇 Markdown 的原生结构，追求高信息密度。

### V. 智性克制 (Skeptical Humility)
- **核心：** 对技术奇点保持警惕，对当下实践保持敬畏。
- **执行：** 结论不给定论，多用设问或开放式结尾。承认技术的局限性，比吹嘘技术的全能更显专业。

## 6. 风格宣言 (Style Manifesto)
...

- **反营销**: 严格避免使用 "game-changing", "revolutionary", "paradigm shift" 等词汇。
- **视觉**: 相比于图库照片，更倾向于使用 **Geek Stick Figures (极客火柴人)** 或结构化的 Mermaid 图表。
- **数据胜过形容词**: 用具体的延迟数值/P99 代替“非常快”。
- **基底**: 将文本视为逻辑的**结构化基底 (Substrate)**，而不仅仅是交流媒介。
