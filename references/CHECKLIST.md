## Content Quality

### Core Requirements
- [ ] **Clear purpose**: The post has a well-defined goal and target audience
- [ ] **Original insight**: Goes beyond summarizing, adds unique perspective
- [ ] **Accurate information**: All claims are factual and verifiable
- [ ] **Proper attribution**: Sources, papers, and references are cited

### Value Proposition
- [ ] **Reader benefit**: Clear what the reader will learn
- [ ] **Actionable takeaways**: Includes practical insights or next steps
- [ ] **Right depth**: Matches promise - not too shallow, not unnecessarily deep

### Completeness
- [ ] **Questions answered**: Addresses the "why", "what", "how"
- [ ] **No loose ends**: Doesn't introduce concepts without explanation
- [ ] **Conclusion wraps up**: Ending provides closure or forward direction

## Structure & Flow

### Opening
- [ ] **Strong hook**: First paragraph engages reader
- [ ] **Clear premise**: Reader knows what to expect
- [ ] **Right length**: Introduction doesn't meander

### Body
- [ ] **Logical progression**: Ideas build on each other
- [ ] **Clear sections**: Headings and subheadings guide reader
- [ ] **Smooth transitions**: Between paragraphs and sections
- [ ] **One idea per paragraph**: No paragraph overcrowding

### Ending
- [ ] **Satisfying conclusion**: Doesn't end abruptly
- [ ] **Key takeaways summarized**: Main points reinforced
- [ ] **Forward-looking**: May suggest next steps or future directions (if appropriate)

## Technical Accuracy

### Code Examples
- [ ] **Code runs**: All examples are tested and functional
- [ ] **Syntax highlighted**: Proper formatting for language
- [ ] **Well commented**: Key sections explained
- [ ] **Complete**: No missing imports or setup steps

### Technical Claims
- [ ] **Verifiable**: Can be backed by sources or testing
- [ ] **Nuanced**: Avoids overgeneralization
- [ ] **Up-to-date**: Reflects current state of technology
- [ ] **Honest limitations**: Acknowledges what approach doesn't do

### Data & Metrics
- [ ] **Specific numbers**: "10x" not "significantly"
- [ ] **Context provided**: What do numbers mean?
- [ ] **Comparisons fair**: Apples-to-apples when comparing
- [ ] **Sources cited**: Where do metrics come from?

## Style Consistency

### Voice & Tone
- [ ] **Consistent voice**: Doesn't shift between formal/casual
- [ ] **Appropriate tone**: Matches content and audience
- [ ] **No jargon overload**: Technical terms explained
- [ ] **Balanced sentence length**: Mix of short and long

### Perspective
- [ ] **Consistent pronouns**: "we" vs "I" vs "you" - stick to one
- [ ] **Clear attribution**: Distinguishes facts from opinions
- [ ] **Appropriate confidence**: Doesn't overstate certainty

### Format
- [ ] **Consistent headings**: H1 → H2 → H3 hierarchy
- [ ] **Consistent lists**: All bullets or all numbers
- [ ] **Consistent emphasis**: Bold/italic used purposefully

## Visual Elements

### Figures & Diagrams
- [ ] **Clear purpose**: Each visual adds value
- [ ] **High quality**: Resolution sufficient for detail
- [ ] **Captioned**: Every figure has descriptive caption
- [ ] **Referenced**: Text mentions figures where relevant

### Code Blocks
- [ ] **Language specified**: Syntax highlighting applied
- [ ] **Proper indentation**: Maintains structure
- [ ] **Line length**: Not too wide (max ~100 chars)
- [ ] **Surrounding context**: Introduced and explained

### Tables
- [ ] **Clear headers**: Column labels are descriptive
- [ ] **Aligned data**: Numbers right-aligned, text left-aligned
- [ ] **Captioned**: Tables have descriptive captions
- [ ] **Not overloaded**: Break complex tables into multiple

### Emojis (if used)
- [ ] **Purposeful**: Each emoji serves a function
- [ ] **Consistent**: Used throughout or not at all
- [ ] **Accessible**: Doesn't rely on emoji for meaning

## Language & Tone

### Clarity
- [ ] **Active voice**: Prefer "We built" over "It was built"
- [ ] **Simple words**: "Use" not "utilize", "help" not "facilitate"
- [ ] **Concrete examples**: Abstract ideas illustrated
- [ ] **No filler**: Removes "very", "really", "quite" where unnecessary

### Style-Specific Checks

**OpenAI Style:**
- [ ] Data-driven: Claims backed by numbers
- [ ] Challenge → Solution format for problems
- [ ] Before/After comparisons for changes
- [ ] Production-reliability focus

**Anthropic Style:**
- [ ] Extensive concrete examples
- [ ] Numbered definitions for key concepts
- [ ] Explicit "Limitations" section
- [ ] "Looking forward" conclusion

**Karpathy Style:**
- [ ] Conversational opening
- [ ] "Tips & tricks" sections
- [ ] Code with "What's happening" blocks
- [ ] Opinionated but fair

**Yi Tay Style:**
- [ ] Personal narrative elements
- [ ] Gratitude/acknowledgements
- [ ] Growth framing
- [ ] Data/numbers sharing

### Grammar & Mechanics
- [ ] **Spelling checked**: No typos
- [ ] **Grammar correct**: Proper tense, subject-verb agreement
- [ ] **Punctuation**: Commas, periods in right places
- [ ] **Capitalization**: Consistent and correct

## Publication Ready

### Metadata
- [ ] **Title**: Clear, descriptive, engaging
- [ ] **Reading time**: Accurately estimated
- [ ] **Date**: Publication date set
- [ ] **Tags/categories**: Appropriate and complete
- [ ] **Author**: Attributed correctly

### Links & References
- [ ] **All links work**: Test each URL
- [ ] **Sources cited**: Proper attribution
- [ ] **Further reading**: Relevant links included
- [ ] **No broken references**: All citations resolve

### Cross-Platform
- [ ] **Mobile friendly**: Reads well on phone
- [ ] **Dark mode**: Images work in both themes
- [ ] **Accessible**: Alt text for images
- [ ] **SEO optimized**: Meta description if applicable

### Final Polish
- [ ] **Read aloud**: Flows smoothly when spoken
- [ ] **Peer reviewed**: Someone else checked it
- [ ] **Fresh eyes**: Reviewed after a break
- [ ] **Version control**: Properly saved/backed up

## Pre-Publish Questions

1. **Would I share this with a colleague?** If no, revise
2. **Will this be relevant in 6 months?** If no, consider if it should be a tweet instead
3. **Did I deliver on the title's promise?** If no, adjust title or content
4. **Is there anything I'd be embarrassed by later?** If yes, remove or rephrase
5. **Would I be proud if this appeared on [OpenAI/Anthropic/Karpathy]'s blog?** If no, polish more

## Quick-Edit Checklist (5-minute review)

- [ ] Scan headings: Are they descriptive and parallel?
- [ ] Check first paragraph: Does it hook?
- [ ] Check last paragraph: Does it land?
- [ ] Look for "very", "really", "quite": Remove unnecessary instances
- [ ] Check all links: Click each one
- [ ] Count the examples: Are there enough? (Target: 2-3 minimum)
- [ ] Look at code blocks: Are they explained?
- [ ] Scan for typos: Use Ctrl+F for common mistakes
- [ ] Check tone: Is it consistent throughout?
- [ ] Verify claims: Can you back them up?

## Common Issues to Watch

### Content Issues
- ❌ **Wall of text**: Break up with headings/paragraphs
- ❌ **Buried lead**: Key insight too deep
- ❌ **No examples**: Abstract without illustration
- ❌ **Abrupt ending**: Leaves reader hanging

### Style Issues
- ❌ **Inconsistent voice**: Shifts between formal/casual
- ❌ **Passive voice**: "It was decided" vs "We decided"
- ❌ **Jargon without explanation**: Assumes too much knowledge
- ❌ **Overstating**: "Best", "Perfect", "Always" - rarely accurate

### Technical Issues
- ❌ **Untested code**: Examples don't run
- ❌ **Dead links**: URLs don't resolve
- ❌ **Wrong attribution**: Mis-cited sources
- ❌ **Outdated info**: Refers to deprecated APIs/methods

### Format Issues
- ❌ **Missing alt text**: Images inaccessible
- ❌ **Broken formatting**: Indentation, spacing issues
- ❌ **Inconsistent headings**: Hierarchy混乱
- ❌ **Poor mobile rendering**: Doesn't work on phone
