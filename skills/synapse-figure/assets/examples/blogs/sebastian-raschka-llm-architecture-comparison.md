# Sebastian Raschka - The Big LLM Architecture Comparison

## Source

- title: The Big LLM Architecture Comparison
- url: https://magazine.sebastianraschka.com/p/the-big-llm-architecture-comparison
- source type: editorial technical blog

## Why It Is Good

- It is dense without becoming unreadable.
- It uses figures to compress architecture differences instead of only decorating the article.
- It alternates between explanation, comparison, and summary well.

## Writing Traits

- title style: broad comparison headline with concrete technical scope
- section rhythm: repeated pattern of topic intro -> figure -> explanation -> local summary
- paragraph density: high, but each paragraph tends to carry one technical point
- caption style: direct and functional
- typography impression: clean editorial long-form, optimized for reading

## Figure Traits

- figure classes used:
  - architecture comparison
  - attention mechanism comparison
  - annotated paper figure reuse
  - benchmark and ablation charts
- color behavior:
  - light backgrounds
  - restrained technical palette
  - color used to separate mechanism roles
- shape and line language:
  - modular blocks
  - side-by-side comparison layouts
  - arrows and grouped containers
- label style:
  - short, technical, explicit
- density:
  - medium to high
- chart style:
  - explanatory, not presentation-heavy

## Reusable Rules

- Use side-by-side comparisons to explain architecture deltas.
- Put a figure immediately before or after the explanation it supports.
- Use local summaries to close dense sections.
- Let figures carry mechanism compression, not just ornament.

## Avoid

- copying every figure style literally from paper sources
- bloating captions
- expanding comparison figures until they become unreadable
