03 / Efficiency

维度：为什么 CLI 胜过 MCP / GUI？

  • 母语: LLM 在海量 man page 中练就了 CLI 直觉。
  • 效率: CLI 是 O(1) Context，省去 50k+ Schema。
  • 架构: 文本进/出；零模态转换，零截屏幻觉。
  • Unix: 管道组合天然适配 Agent 逻辑流。
$ cat logs.txt | grep "ERROR" # 1.2k tokens
vs 
MCP Accessibility Tree # 52k tokens
「 简洁是 Token 效率的唯一救赎。 」
whyj + nano banano + 2026/03/30
04 / Evolution

范式转移：Code-as-Action 的崛起

  • 传统模式: 预定义 Tool 列表（易错且昂贵）。
  • 新范式: 模型直接写代码（bash）发现并执行。
  • 准确率: 减少工具干扰后，胜率可提升至 100%。
  • 本质: CLI 让 Agent 从“猜接口”进化为“做工程”。
REACT LOOP:
Reason → Act (CLI)
→ Observe → Reason
「 复杂的工具列表是 Agent 的杂讯。 」

Built for AI Agents: 飞书与钉钉
