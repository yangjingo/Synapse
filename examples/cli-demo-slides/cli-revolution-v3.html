03 / Mechanism

母语优势：LLM 的 Unix 哲学复兴

  • 训练分布: LLM 在海量源码与文档中练就了极强的 CLI 直觉。
  • 管道编排: Unix 管道哲学天然适配 Agent 的多步逻辑编排流。
  • 确定性: 纯文本 I/O 彻底消除截屏识别幻觉与模态转换损耗。
「 简洁是 Token 效率的唯一救赎。 」

04 / Mechanism

ACI：从猜接口到做工程

  • Tool-calling 局限: 预定义静态列表昂贵且容易引发幻觉。
  • Code-as-Action: 模型直接编写动态脚本（Bash/Python）来发现并操作。
  • 状态闭环: CLI 提供了 Agent 推理所需的“非黑即白”的硬反馈。
「 复杂的工具列表是 Agent 的杂讯。 」
AGENTIC LOOP:
Reason → Code (CLI) → Observation → Reason

Built-for-Agent：飞书与钉钉的实践

  • 自愈架构: 提供结构化报错，引导 Agent 实时自动修复参数。
  • 预演模式: 内置 --dry-run，注入工程安全基因。
  • 动态发现: 拒绝硬编码，适配 AI 的探索式交互逻辑。

确定性验证：State-check 胜过 Vibe-check

  • 难题: 传统的 Chat 模式只能靠“感觉”来判断好坏。
  • 验证源: 通过 ==Exit Code== 和 ==File State== 进行硬核验证。
  • 唯一指标: 成功率（Pass@1）是衡量 Agent 推理能力的唯一硬通货。
