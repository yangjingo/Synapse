03 / Mechanism

三种后训练，三种学习问题

  • SFT: 外部给定轨迹 → Forward KL → 监督最稠密，但 train/inference 分布不一致
  • RL: 自身策略轨迹 → Reverse KL → 反馈最稀疏，但优化的是解集合
  • OPD: 学生生成轨迹 + 教师 token 级对齐 → On-policy + Dense signal
「 SFT 记忆轨迹，RL 搜索策略，OPD 对齐分布。 」
whyj + OPD + 2026/04/29
04 / Mechanism

Forward KL vs Reverse KL：投影几何决定泛化

  • Forward KL (SFT): D_KL(q ‖ π_θ) → Mass-covering → 不遗漏教师概率 → 窄监督下退化为拟合单条轨迹
  • Reverse KL (RL/OPD): D_KL(π_θ ‖ p_T) → Mode-seeking → 允许忽略尾部 → 多最优解时偏向 KL-minimal
  • KL-regularized RL 最优解 = reward-tilted policy，不是脱离初始策略的全新分布
「 Forward KL 覆盖，Reverse KL 收缩。方向选错，泛化就崩。 」
Chan et al. - Rethinking KL Divergence

05 / Mechanism

OPD 为什么有效：Dense Signal + On-Policy

  • SFT-2M 外推 70% AIME'24 ≈ 3.4×10²¹ FLOPs
  • RL 达到 67.6% ≈ 17,920 GPU hours
  • OPD 达到 74.4% 仅需 1,800 GPU hours（RL 的 1/10）
  • Self-distillation: OPD 用 7-10x 更少梯度步数复制 RL 策略 → 累计 50-100x
「 OPD 不是压缩 RL 的结果，而是抄捷径学会 RL 搜索到的策略。 」

06 / Mechanism

OPD 为什么会崩：强老师悖论

  • 反直觉: 更大更强的老师 → 蒸馏效果反而更差，甚至回退到 pre-RL 水平
  • Thinking pattern 必须对齐: 教师推理风格与学生差距过大 → 覆盖学生有效结构
  • 局部信息量决定成败: 两个教师能力差距大，但在 student-visited states 上局部分布相似 → 蒸馏效果几乎一样
  • 高熵 mode collapse: 纯 Reverse KL 在教师高熵位置压缩多样性 → Entropy-Aware OPD 需引入 Forward KL
「 决定蒸馏效果的，不是教师有多强，而是在学生真正到达的地方有多准。 」
Teacher Score: 95%
Student Match: 12%
→ Distillation FAIL

Teacher Score: 72%
Student Match: 89%
→ Distillation OK
Rethinking OPD - Tsinghua 2026

07 / Mechanism

OPD 的工程定位：胶水层

  • SFT 打地基: 注入领域知识，建立基础行为
  • RL 精调策略: 在解空间中搜索语义策略
  • OPD 做桥接: 稠密信号将教师能力迁移给学生，成本仅为 RL 的 1/10
  • Continual Learning: SFT 在自身样本上仍退化；OPD 教师固定 → 自蒸馏不退化
「 OPD 是后训练管线的胶水——连接知识注入与行为塑形。 」
SFT → OPD → RL
↑ 又来一轮 ↑
Mid-train → OPD → Deploy
↑ 持续迭代 ↑

08 / Practice

DeepSeek V4：多专家 OPD 替代 Mixed RL

  • V3.2: 专家生成数据 → Student SFT → Mixed GRPO RL 补偿（off-policy）
  • V4: 10+ 领域专家各自 SFT+GRPO → Student rollout → 多教师在学生轨迹上逐 token 反馈（on-policy）
  • Reverse KL 保护学生已有能力：student 不采样到的 mode 不付代价 → 领域特化不互相稀释
  • Rethinking OPD 实测：共享 base 的 teacher（overlap 高）效果远胜 benchmark 更高但 pattern 不匹配的 teacher
「 V4 把 OPD 从"蒸馏工具"升级为"专家融合管线"——RL 退居上游专家训练，OPD 接管整合。 」
V3.2
Experts → Data → SFT → GRPO RL
off-policy, forward KL + RL 补

V4
Experts → Student Rollout → Multi-Teacher OPD
on-policy, reverse KL, 无需 RL 补偿

09 / Insight

反观人类学习：学校就是多专家 OPD

  • 学英语的比喻: 跟带方言口音的老师学语法 — Forward KL 让你连口音一起学，Reverse KL 只在你开口后纠正语法
  • 学霸模式 = Pre-train + OPD: 先听课拉齐分布，再自己做题，带着卡点问老师 — 在自己生成的轨迹上拿反馈
  • OPD 的前提: Forward KL 冷启动（认真预习）→ 把 student 拉到能听懂 teacher 反馈的位置 → 然后才能 on-policy
  • LLM 没有改写学习本质: 它只是把 reverse KL 所需的"高质量专家反馈"从稀缺资源变成了随手可得
「 先用 Forward KL 把自己拉到能听懂反馈的地方，再用 Reverse KL 在自己的轨迹上精修。 」
学习模式映射

Pre-train = 泛读建立基础
SFT = 听课拉齐分布
RL = 硬刚题目（高耗低效）
OPD = 带着卡点问老师
← 最优解

10 / Takeaways

三条铁律

  • KL 方向是 OPD 的命门 — Reverse KL 的 mode-seeking 在低熵位置是优势，在高熵位置是毒药
  • 教师强度 ≠ 蒸馏质量 — 成败取决于教师在 student-visited states 上的局部信息量，而非全局 benchmark 分数
  • OPD 是 dense-constrained RL 的特例 — 不是"廉价 RL"，而是有独立理论结构和独立失效模式的方法
「 把 OPD 当"便宜 RL"用，是你训崩的开始。 」
