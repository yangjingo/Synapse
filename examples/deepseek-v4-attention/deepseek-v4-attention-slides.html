07 / Mechanism II

标准注意力计算：Scaled Dot-Product Attention

  • 公式: S = Q @ K^T → softmax → P @ V
  • 参数: mask 预留，用于控制 KV 的选择范围。
  • 复杂度: 全局 O(L^2)，所有 KV 参与计算。 
def ScaledDotProductionAttention(Q, K, V, M):
    S = Q @ K.transpose(-2, -1)
    if M != None:
        S = S.masked_fill(M == False, float('-inf'))
    P = F.softmax(S, dim = -1)
    Z = P @ V
    return Z

Mask-Free：SingleQuerySDPA 实现

  • 单 query 接口: 输入一个 query 向量和 K、V 子集，输出注意力结果。
  • 基本计算单元: 这是所有稀疏注意力的基础。 
def SingleQuerySDPA(q, K, V):
    S = q @ K.t()
    P = F.softmax(S, dim=-1)
    Z = P @ V
    return Z

11 / Mechanism II

Mask-Free：完整遍历实现

class AttentionFreeMask(nn.Module):
  def __init__(self, args):
    super().__init__()
    self.Wq = nn.Linear(dim, n_heads * head_dim, bias=False)
    self.Wk = nn.Linear(dim, n_kv_heads * head_dim, bias=False)
    self.Wv = nn.Linear(dim, n_kv_heads * head_dim, bias=False)
    self.Wo = nn.Linear(dim, dim, bias=False)

  def forward_free_mask(self, X, mask):
    B, L, D = X.shape
    Q, K, V = self.Wq(X), self.Wk(X), self.Wv(X)
    Z = torch.zeros_like(Q)
    for i in range(B):
      for j in range(L):
        idx = torch.where(mask[i, j] != 0)[0].tolist()
        q = Q[i,[j], :]
        K_ = K[i, idx, :]
        V_ = V[i, idx, :]
        Z[i,j,:] = SingleQuerySDPA(q, K_, V_)
    O = self.Wo(Z)
    return O
  • 逐 batch + 逐 query: 双层循环，对每个 query 找到 mask 非零位置。
  • 只计算必要 KV: 真正减少计算量。
  • 输出投影: 最后通过 Wo 映射回原始维度。

Top-K 注意力：Post vs Pre 模式

  • Post: 先计算完整注意力分数，再选 Top-K。无法避免 O(L^2)。
  • Pre: 由 Indexer 先行选 KV，再计算注意力，只算必要位置。

13 / Mechanism II

Top-K：Pre 模式为什么要先选？

  • Post 的问题: 必须先算全量 S = Q @ K^T，O(L^2)，top-k 只是后处理。
  • Pre 的优势: 先用轻量评分筛选 KV，只对选中的计算注意力。
  • V4 的选择: Pre 模式——Indexer 先选 KV，再逐 query 计算。

class AttentionTopK(nn.Module):
  def forward(self, X, topk=10, is_post=True):
    B, L, D = X.shape
    Q, K, V = self.Wq(X), self.Wk(X), self.Wv(X)
    Z = torch.zeros_like(Q)
    for i in range(B):
      for j in range(L):
        if is_post:
          S = Q[i,[j], :] @ K[i].t()
          # ... select top-k from S
        else:
          # Pre: Indexer 先选 KV
          S = torch.randn(1, L)  # 模拟评分
          _, idx = torch.topk(S, topk)
          idx = idx[0].tolist()
          q = Q[i,[j], :]
          K_ = K[i, idx, :]
          V_ = V[i, idx, :]
          Z[i,j,:] = SingleQuerySDPA(q, K_, V_)
    O = self.Wo(Z)
    return O

  • 零浪费: 只对选中的 top-k KV 计算注意力。

17 / Mechanism III

压缩注意力：代码实现

class AttentionCompress(nn.Module):
  def forward(self, X, compress_ratio):
    B, L, D = X.shape
    Q, K, V = self.Wq(X), self.Wk(X), self.Wv(X)
    c, n_c = compress_ratio, L // compress_ratio
    if compress_ratio != 0:
      # mean-pooling 压缩
      K_ = K.reshape(B, n_c, c, D).mean(dim=2)
      V_ = V.reshape(B, n_c, c, D).mean(dim=2)
    Z = ScaledDotProductionAttention(Q, K_, V_, None)
    O = self.Wo(Z)
    return O
  • reshape + mean: 按 compress_ratio 分块取平均。
  • 复用标准注意力: 压缩后的 KV 直接用 ScaledDotProduct。

滑窗注意力：代码实现

class AttentionSlidingWindow(nn.Module):
  def forward(self, X, window_size):
    B, L, D = X.shape
    Q, K, V = self.Wq(X), self.Wk(X), self.Wv(X)
    Z = torch.zeros_like(Q)
    for i in range(B):
      for j in range(L):
        left = max(j, j - window_size)
        right = min(j, j + window_size)
        q = Q[i,[j], :]
        K_ = K[i, left:right, :]
        V_ = V[i, left:right, :]
        Z[i,j,:] = SingleQuerySDPA(q, K_, V_)
    O = self.Wo(Z)
    return O
  • 窗口裁剪: left/right 边界确定窗口范围。
  • 输出: torch.Size([2, 256, 512])

23 / Mechanism IV

HCA：Compressor 类设计

class Compressor(nn.Module):
  def __init__(self, compress_ratios, dim):
    super().__init__()
    self.compress_ratios = compress_ratios
    self.w_compress = nn.Linear(
        compress_ratios, 1, bias=False)

  def forward(self, X):
    B, L, D = X.shape
    c, n_c = self.compress_ratios, L // c
    if self.compress_ratios > 1:
      X_ = X.reshape(B, n_c, c, D)
      X_ = self.w_compress(
          X_.transpose(2,3))[...,0]
    return X_
  • 可学习压缩: 线性投影 w_compress 做加权压缩。
  • 对比 mean-pooling: 表达能力更强。
  • 参数: w_compress [ratio, 1]。

HCA：信息泄漏分析

  • 问题: 一个压缩块包含多个 token 的 KV，可能包含未来信息。

CSA：Indexer 类设计

class Indexer(nn.Module):
  def __init__(self, args):
    super().__init__()
    dim = args.dim
    # 简化版：投影到1维做点积评分
    self.Wk_light = nn.Linear(dim, 1, bias=False)
    self.Wq_light = nn.Linear(dim, 1, bias=False)

  def forward(self, Q, K, topk):
    Q_ = self.Wq_light(Q)
    K_ = self.Wk_light(K)
    S = Q_ @ K_.transpose(-2, -1)
    _, idx = torch.topk(S, topk, dim=-1)
    return idx
  • 轻量投影: 映射到 1 维做点积评分。
  • V4 实际: 64 维多头评分 + 可学习加权。
  • topk 选出: 稀疏索引。

CSA：前向计算流程

  • Step 1: Compressor 对 KV 做轻量压缩（ratio=4）。

CSA：Overlap 代码实现

def overlap(X):
  B, L, D = X.shape
  X_ = torch.zeros(B, L, D*2)
  X_[:, :, D:] = X            # 当前段
  X_[:, 1:, :D] = X[:, :L-1, :]  # 前一段
  return X_

overlap_flag = 1
coff = 1 + overlap_flag

K_ = overlap(compressor(K))
V_ = overlap(compressor(V))

if overlap_flag:
  _, L_c, _ = K_.shape
  K_ = K_.reshape(bsz, L_c, args.dim, coff).sum(dim=-1)
  V_ = V_.reshape(bsz, L_c, args.dim, coff).sum(dim=-1)
  • overlap 拼接: 当前段 + 前一段 → 2D。

class MQA(nn.Module):
  def __init__(self, args, o_lora_rank, o_groups):
    super().__init__()
    dim = args.dim
    head_dim = args.head_dim
    self.n_heads = args.n_heads
    self.n_kv_heads = 1
    self.Wq = nn.Linear(dim,
      self.n_heads * head_dim, bias=False)
    self.Wc = nn.Linear(dim, head_dim, bias=False)
    self.Wk = nn.Linear(head_dim, head_dim, bias=False)
    self.Wv = nn.Linear(head_dim, head_dim, bias=False)
    self.Wo_a = nn.Linear(
      self.n_heads * head_dim // o_groups,
      o_groups * o_lora_rank, bias=False)
    self.Wo_b = nn.Linear(
      o_lora_rank * self.n_heads, dim, bias=False)
  • Wc: 下投影到单头。
  • Wk/Wv: 单头 KV 投影。
  • Wo_a/Wo_b: 分组输出投影。

V4-MQA：前向计算代码

  def forward(self, X, mask):
    B, L, D = X.shape
    Q = self.Wq(X)
    K = self.Wk(self.Wc(X))
    V = self.Wv(self.Wc(X))
    Q = Q.reshape(B, L, self.n_heads,
              -1).transpose(1, 2)
    K = K.reshape(B, L, 1, -1).transpose(1, 2)
    V = V.reshape(B, L, 1, -1).transpose(1, 2)
    Z = ScaledDotProductionAttention(
        Q, K, V, mask)
    d = self.n_heads * D // self.o_groups
    Z = Z.reshape(B, L, self.o_groups, d)
    Z = self.Wo_a(Z)        # 分组降维
    Z = Z.reshape(B, L, -1)
    O = self.Wo_b(Z)        # 升维回 dim
    return O
  • 注意: K/V reshape 为 n_kv_heads=1。
  • Wo_a: 分组降维。
  • Wo_b: 升维回 dim。

Group Projection：五条洞察

  • 成组方式: 每 8 个注意力头拼为一条组向量，128 头共得 16 条。
  • 组隔离: 每条组向量只与 Wo_a 中对应子矩阵交互。
  • 权重独立: 128 个头压成 16 个"共享组头"，每组独立权重。
  • 计算量可控: 总开销由组数、组内维度与中间维度决定。
  • 硬件友好: 贴合 Tensor Core，适配 FP4/FP8 量化。
「 分组不是简单拆矩阵——是把 128 个头压成 16 个共享组头。 」

Sparse Kernel：逐 query 块式 gather

def sparse_linear_attn_torch(
    q, kv, topk_ids, block=64):
  L, H, d = q.shape
  K_topk = topk_ids.shape[1]
  num_blocks = math.ceil(K_topk / block)
  o = torch.zeros(L, H, d)

  for i in range(L):         # 逐 query
    qi = q[i]                 # [H, d]
    acc_o = torch.zeros(H, d)
    for t in range(num_blocks): # 分块
      start = t * block
      end = min(start + block, K_topk)
      idxs = topk_ids[i, start:end]
      kv_block = kv[:, idxs]  # 非连续
      scores = qi @ kv_block[0].T
      acc_o += scores @ kv_block[1]
    o[i] = acc_o
  return o
  • qi @ K_block.T: [H, d] @ [d, blk] = [H, blk]
  • score @ V: [H, blk] @ [blk, d]
  • 累加: 跨块累加。

Sparse Kernel：三个关键提炼

49 / Summary

V4 注意力方案全景回顾

  • HCA: block-level 强压缩（ratio=128），负责全局信息理解。
  • CSA: token-level 稀疏选取 + 轻量压缩（ratio=4），负责精确注意力建模。
  • SWA: 局部窗口（128 token），近场兜底。
  • MQA: 单头 KV + 分组投影，减少 KV-Cache + 控制输出参数。
  • Sparse Kernel: 逐 query 块式 gather，零浪费计算。
「 block-level 全局 + token-level 精确 + 局部窗口兜底。 」

51 / Take A Ways

Take A Ways

  • Q1 → A1: 标准注意力 → Mask-Free → Top-K → 压缩 → 滑窗，每层解决一个具体问题。
  • Q2 → A2: Mask-Free 逐 query 拉取真正减少计算；Top-K Pre 模式先选后算避免 O(L^2)。
  • Q3 → A3: 压缩注意力的数学直觉是"距离越远压缩越狠"，工程限制由 HCA/CSA 各自解决。
  • Q4 → A4: Compressor 用可学习投影做因果强压缩，信息空缺由 SWA 填补。
  • Q5 → A5: Indexer 保证精度，Compressor 弥补全局，overlap 消除断裂，统一 Softmax。
  • Q6 → A6: Wo 拆成 Wo_a + Wo_b，128x512=65536 → 参数量从 O(D^2) 降到 O(D*r)。
「 V4 注意力方案整体是 NSA + DSA 的有机融合——block-level 全局 + token-level 精确 + 局部窗口兜底，Sparse Kernel 逐 query 零浪费计算。 」
